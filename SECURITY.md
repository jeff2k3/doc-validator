# Security Policy

## Reporting a Vulnerability

> [!WARNING]
>**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, use one of the following methods:

### Option 1: GitHub Security Advisory ()Recommended
1. Go to [https://github.com/jeff2k3/doc-validator/security](https://github.com/jeff2k3/doc-validator/security)
2. Click on **"Report a vulnerability"**.
3. Fill in the details (version, description, steps to reproduce).

### Option 2: Email
If you prefer, you can send an email with:

- Version of the library you are using.
- A detailed description of the vulnerability (how to exploit, impact).
- Your GitHub username (optional) if you wish to be credited.

Please note: we cannot guarantee a reply to every email, but we will acknowledge receipt as soon as possible.

---

## FAQ

### Will I be credited?
Yes, if you wish. We will include your name/handle in the security advisory (with your consent).
